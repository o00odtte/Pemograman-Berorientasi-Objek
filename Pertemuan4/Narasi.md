# Refleksi Praktikum: Naming Convention dan Operator Aritmatika

Pada praktikum ini saya belajar tentang naming convention dan penggunaan operator aritmatika dalam Java. Dari latihan yang dikerjakan, saya mulai memahami bahwa menulis program tidak hanya soal bisa dijalankan, tetapi juga bagaimana kode tersebut mudah dibaca dan dipahami, baik oleh diri sendiri maupun orang lain.

Naming convention sendiri saya pahami sebagai aturan atau kebiasaan dalam memberi nama pada variabel, class, dan bagian lain dalam program. Misalnya, class menggunakan huruf kapital di awal (PascalCase), sedangkan variabel menggunakan camelCase. Awalnya saya merasa penamaan tidak terlalu berpengaruh, yang penting program jalan. Namun setelah mencoba beberapa latihan, saya sadar bahwa penamaan yang rapi sangat membantu dalam memahami isi program, apalagi kalau kodenya mulai banyak.

Menurut saya, penamaan variabel dan class itu penting karena berhubungan langsung dengan keterbacaan kode. Kalau nama variabel terlalu singkat atau tidak jelas, kita jadi harus membaca ulang logika program untuk memahaminya. Sebaliknya, kalau namanya jelas, kita bisa langsung tahu fungsinya. Selain itu, penamaan yang konsisten juga membuat kode terlihat lebih rapi dan enak dibaca.

Saya juga mempelajari cara kerja operator aritmatika dalam Java, seperti penjumlahan, pengurangan, perkalian, pembagian, dan modulus. Selain itu, ada juga operator increment (++) dan decrement (--), serta operator seperti +=, -=, dan *=. Dari latihan kalkulator dan counter, saya jadi lebih paham bagaimana operator tersebut digunakan untuk mengubah nilai variabel secara bertahap.

Contoh yang paling saya pahami adalah saat mengerjakan latihan produk dan kalkulator. Pada latihan produk, saya belajar bagaimana menggunakan variabel dengan tipe data berbeda dalam satu program, seperti String, int, boolean, dan double. Dari situ saya jadi lebih paham bahwa setiap tipe data punya peran masing-masing dan harus digunakan dengan tepat, terutama saat melakukan perhitungan seperti total nilai dan pajak. Sedangkan pada latihan kalkulator, saya lebih memahami cara kerja operator aritmatika secara langsung, mulai dari penjumlahan, pengurangan, perkalian, pembagian, hingga modulus. Dari latihan ini saya juga mulai sadar bahwa tipe data sangat berpengaruh pada hasil operasi, misalnya pembagian bilangan bulat tidak menghasilkan angka desimal. Latihan ini membantu saya memahami dasar operasi dalam Java secara lebih nyata karena hasilnya bisa langsung dilihat dari output program.

Kesulitan yang saya alami ada di beberapa bagian. Pertama, saat memahami tipe data dalam operasi aritmatika, terutama pembagian. Saya sempat bingung kenapa hasilnya tidak sesuai, ternyata karena masih menggunakan tipe integer. Selain itu, saya juga agak bingung di bagian counter, terutama saat memisahkan proses perhitungan dengan bagian output. Saya sempat ragu apakah nilai harus langsung diubah atau disimpan ke variabel baru. Tapi setelah mencoba beberapa kali, saya mulai memahami alurnya dan lebih terbiasa membaca perubahan nilai secara bertahap.

Secara keseluruhan, praktikum ini membantu saya memahami dasar pemrograman Java dengan lebih jelas. Saya tidak hanya belajar menulis kode, tetapi juga mulai memperhatikan cara menulis program yang rapi, mudah dibaca, dan logis.
